General questions¶

  1. What is anaconda?

    Anaconda is a meta package for conda. This meta package contains all standard packages (like scipy, numpy, zlib etc.) provided by developers from Continuum Analytics.

Getting help¶

  1. How can I see what conda commands are supported?

    $ conda -h

  2. How can I get help for X command (e.g. create)?

    $ conda create -h

Searching & info¶

  1. How can I find out what version of conda I have installed?

    By typing:

    $ conda info

    or by typing:

    $ conda -V

  2. How can I check if package X (e.g. scipy) is available for installation?

    $ conda search scipy

  3. How can I check if package X (e.g. scipy) is available for installation from the Continuum repos (i.e., from Anaconda)?

    $ conda search --override-channels --channel defaults scipy

  4. How do I check if package X (e.g. iminuit) exists in channel Y (e.g. http://conda.binstar.org/mutirri) and is available for installation?

    $ conda search --override-channels --channel http://conda.binstar.org/mutirri iminuit

Customization (.condarc file)¶

Note

Sometimes to perform the below commands it is necessary to add the -f option (aka –force).

  1. How can I get all keys and their values from my .condarc file?

    $ conda config --get

  2. How can I get the value of key X (e.g. channels) from my .condarc file?

    $ conda config --get channels

  3. How can I add a new value Y (e.g. http://conda.binstar.org/mutirri) to key X (e.g. channels)?

    $ conda config --add channels http://conda.binstar.org/mutirri

  4. How can I remove existing value Y (e.g. http://conda.binstar.org/mutirri) from key X?

    $ conda config --remove channels http://conda.binstar.org/mutirri

  5. How can I remove the key X (e.g. channels) and all of its values?

    $ conda config --remove-key channels

Package installation (in the root environment)¶

  1. How can I install package X (e.g. scipy)?

    $ conda install scipy

  2. How can I install package X (e.g. scipy) in specific Z.Z.Z version (0.12.0)?

    $ conda install scipy=0.12.0

  3. How can I install many packages at once, like X1 (e.g. scipy) and X2 (e.g. curl)?

    $ conda install scipy curl

  4. How can I install many packages at once, like X1 (e.g. scipy) in version Z.Z.Z (e.g. 0.12.0) and X2 (e.g. curl) in version A.A.A (e.g. 7.26.0)?

    $ conda install scipy=0.12.0 curl=7.26.0

Updating packages (in the root environment)¶

  1. How can I update conda itself?

    $ conda update conda

  2. How do I update the anaconda meta package?

    $ conda update conda
$ conda update anaconda

  3. How can I update package X (e.g. scipy)?

    $ conda update scipy

  4. How can I keep a package from being updated?

    You can pin a package specification in an environment, which will prevent it from being changed, unless the --no-pin flag is passed to conda. To do so, add a file called pinned to the environment’s conda-meta directory with a list of specs. For example

    numpy 1.7.*
scipy ==0.13.2

    will force numpy to stay on the 1.7 series (any version that starts with “1.7.”), and scipy to stay at exactly version 0.13.2.

    With this pinned file, conda update numpy will keep numpy at 0.7.1, and conda install scipy=0.14.0 will lead to an error. To force either of these, use the -no-pin flag, like conda update numpy --no-pin. The way pinning works is that the pined specs are included with each conda install, so subsequent commands without --no-pin will revert numpy back to the 1.7 series.

Removing packages (from the root environment)¶

  1. How can I remove package X (e.g. scipy)?

    $ conda remove scipy

  2. How can I remove multiple packages at once, like X1 (e.g. scipy) and X2 (e.g. curl)?

    $ conda remove scipy curl

Environments¶

Getting info about environments¶

  1. How can I get a list of all of my environments?

    $ conda info -e

  2. How can I list all installed packages in environment X (e.g. myenv)?

    If myenv is not activated:

    $ conda list -n myenv

    If myenv is activated:

    $ conda list

  3. How can I check if package Y (e.g. scipy) is already installed in existing environment X (e.g. myenv)?

    • the first method:

      $ conda list -n myenv scipy

    • the second method:

      $ source activate myenv
$ conda list scipy

Creating new environments¶

  1. How can I create a new and clean environment X (e.g. myenv)?

    $ conda create -n myenv python

  2. How can I create a new environment X (e.g. myenv) with python Y.Y.Y (e.g. 3.4) as the default interpreter inside it?

    $ conda create -n myenv python=3.4

  3. How can I create a new environment X (e.g. myenv) with package Y inside it (e.g. scipy)?

    • in a single command:

      $ conda create -n myenv scipy

    • with more typing:

      $ conda create -n myenv python
$ conda install -n myenv scipy

    • the longest version (also activates the newly created environment):

      $ conda create -n myenv python
$ source activate myenv
$ conda install scipy

  4. How can I create a new environment X (e.g. myenv) with package Y (e.g. scipy) in version Z.Z.Z (e.g. 0.12.0) inside it?

    • in a single command:

      $ conda create -n myenv scipy=0.12.0

    • with more typing:

      $ conda create -n myenv python
$ conda install -n myenv scipy=0.12.0

    • the longest version (activating newly created environment also):

      $ conda create -n myenv python
$ source activate myenv
$ conda install scipy=0.12.0

Activating and deactivating¶

  1. How can I activate the existing environment X (e.g. myenv)?

    $ source activate myenv

  2. How can I deactivate the active environment X (e.g. myenv)?

    $ source deactivate

Installation¶

  1. How can I install package Y (e.g. scipy) in existing environment X (e.g. myenv)?

    • first possibility:

      $ conda install -n myenv scipy

    • the alternate way:

      $ source activate myenv
$ conda install scipy

  2. How can I install Z.Z.Z (e.g. 0.12.0) version of package Y (e.g. scipy) in existing environment (e.g. myenv)?

    If myenv not activated:

    $ conda install -n myenv scipy=0.12.0

    If myenv is activated:

    $ conda install scipy=0.12.0

  3. How can I use pip in my environment X (e.g. myenv)?

    $ conda install -n myenv pip
$ source activate myenv
$ pip <pip_subcommand>

  4. How can I automatically install pip during creation of any of new environment?

    $ conda config --add create_default_packages pip

    After performing the above command you can create a new environment in the standard way (pip will be installed in all of them).

  5. How can I automatically install Y package (e.g. scipy) during creation of any of new environment?

    $ conda config --add create_default_packages scipy

    After performing the above command you can create a new environments in the standard way (the scipy will be installed in all of them).

  6. How can I automatically install version Z.Z.Z (e.g. 0.12.0) of package Y (e.g. scipy) during creation of any of new environment?

    $ conda config --add create_default_packages scipy=0.12.0

    After performing the above command you can create a new environments in the standard way (the scipy in 0.12.0 version will be installed in all of them).

  7. How can I ignore packages from automatic installation during creation of new and clean environment X (e.g. myenv)?

    $ conda create --no-default-packages -n myenv python

Removing¶

  1. How can I remove package Y (e.g. scipy) in existing environment X (e.g. myenv)?

    If myenv is not activated:

    $ conda remove -n myenv scipy

    If myenv is activated:

    $ conda remove scipy

  2. How can I remove existing environment X (e.g. myenv)?

    $ conda remove -n myenv --all

Recipes¶

  1. How can I create a skeleton conda recipe for package X (e.g. bottle) if I know that this package is on PyPI?

    $ conda skeleton pypi bottle

    You can then build it with

    $ conda build bottle

    It is recommended to upload the package to binstar when you are done. Then if you add your binstar channel to your .condarc (see Customization (.condarc file) above), you will be able to install the package with

    $ conda install bottle

    • If you did not upload the package to binstar, to install this package in the root environment, you need to find out where the built package is:

      $ conda info

    • get the root environment path and perform the installation:

      $ conda install <path_from_root_environment_variable>/conda-bld/<your_platform>/bottle.tar.bz2

    This information is also shown at the end of the build process.
